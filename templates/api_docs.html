API Documentation

Complete reference for the WikiDelve REST API. All endpoints accept and return JSON unless noted otherwise.

Base URL: All endpoints are unauthenticated. Content-Type for POST requests: application/json

Overview

The API is organized into sections:

  • Articles — CRUD operations on wiki articles across knowledge bases
  • Search — Full-text search (FTS5) and hybrid vector+graph search
  • Research — Trigger AI research jobs, track progress, manage history
  • Quality — Article scoring, enrichment, crosslinking, fact-checking, freshness audits
  • Knowledge Graph — Entity graph data, relationship traversal
  • Palace — Topic clustering and memory palace navigation
  • Ingest — Import from YouTube, PDFs, ePubs, and web directories
  • GitHub — Track releases and index repositories
  • System — Health checks and statistics

Async operations (research, quality, ingest) return a {"status": "queued"} response and process in the background via worker queue. Poll status endpoints to track progress.

Articles

Read, list, and delete wiki articles stored in markdown knowledge bases.

GET /api/articles List all articles

Query Parameters

NameTypeDescription
kbstring optionalFilter by knowledge base name (e.g. personal, research)
sourcestring optionalFilter by source type: local, web, media

Response

[
  {
    "slug": "kubernetes-networking",
    "title": "Kubernetes Networking",
    "kb": "personal",
    "summary": "Overview of CNI plugins and service mesh...",
    "tags": ["kubernetes", "networking"],
    "word_count": 1420,
    "source_type": "web",
    "created": "2026-04-01T12:00:00Z"
  }
]

Example

curl "/api/articles?kb=personal&source=web"

Search

Full-text and hybrid vector search across all knowledge bases.

GET /api/search Full-text search (FTS5)

Query Parameters

NameTypeDescription
qstring requiredSearch query

Response

[
  {
    "slug": "kubernetes-networking",
    "kb": "personal",
    "title": "Kubernetes Networking",
    "snippet": "...CNI plugins handle <mark>networking</mark>...",
    "rank": -12.5
  }
]

Example

curl "/api/search?q=kubernetes+networking"
GET /api/search/hybrid Hybrid search (FTS5 + vector similarity + knowledge graph)

Query Parameters

NameTypeDescription
qstring requiredSearch query
kbstring optionalFilter by knowledge base
limitint optionalMax results (default: 15)

Response

[
  {
    "slug": "kubernetes-networking",
    "kb": "personal",
    "title": "Kubernetes Networking",
    "score": 0.87,
    "sources": ["fts", "vector", "graph"]
  }
]
POST /api/search/reindex Rebuild the FTS search index

Request Body

None required.

Response

{"status": "indexed", "articles": 142}

Research

Trigger AI-powered research jobs that search the web, synthesize findings, and create wiki articles.

POST /api/research Start a new research job

Request Body

NameTypeDescription
topicstring requiredResearch topic (min 10 characters)
{"topic": "Kubernetes gateway API vs ingress controllers"}

Response

{"job_id": 42, "status": "queued", "topic": "Kubernetes gateway API vs ingress controllers"}

Errors

400 Topic too short or missing
429 Topic was researched within the last 7 days (cooldown)

POST /api/research/local Research from local files/repos (no API keys needed)

Request Body

NameTypeDescription
topicstring requiredResearch topic (min 3 characters)
pathstring requiredLocal filesystem path to scan
patternstring optionalGlob filter (e.g. *.py, *.md)
{"topic": "auth flow", "path": "/code/myproject", "pattern": "*.py"}

Response

{"job_id": 43, "status": "queued", "topic": "auth flow", "path": "/code/myproject"}
GET /api/research/status/{job_id} Poll job status

Path Parameters

NameTypeDescription
job_idint requiredJob ID from research creation

Response

{
  "id": 42,
  "topic": "Kubernetes gateway API",
  "status": "complete",
  "created_at": "2026-04-08T10:00:00Z",
  "completed_at": "2026-04-08T10:02:30Z",
  "articles_created": 3,
  "word_count": 4200,
  "error": null
}

Status values: queued → searching → searching_round_1 → searching_round_2 → synthesizing → complete | error | cancelled

Errors

404 Job not found

GET /api/research/jobs List recent research jobs (last 50)

Response

[
  {"id": 42, "topic": "...", "status": "complete", "created_at": "..."},
  {"id": 41, "topic": "...", "status": "error", "error": "..."}
]
GET /api/research/history Full research history (jobs + article updates)

Response

{
  "jobs": [{"id": 42, "topic": "...", "status": "complete"}],
  "article_updates": [{"slug": "...", "kb": "...", "action": "created"}]
}
GET /api/research/sources/{job_id} Get sources used for a research job

Response

{
  "job_id": 42,
  "sources": [
    {"url": "https://example.com/article", "title": "...", "word_count": 800}
  ]
}

Errors

404 Job not found

POST /api/research/cancel/{job_id} Cancel a queued or in-progress job

Response

{"job_id": 42, "status": "cancelled"}

Errors

404 Job not found

POST /api/research/retry-errors Re-queue all errored jobs

Request Body

NameTypeDescription
limitint optionalMax jobs to retry (default: 50)

Response

{"retried": 3, "total_errors": 5}
DELETE /api/research/{filename} Delete a research output file

Path Parameters

NameTypeDescription
filenamestring requiredResearch file name (no path separators allowed)

Response

{"status": "deleted", "filename": "research-output.md"}

Errors

400 Invalid filename (contains /, \, or ..)
404 File not found

DELETE /api/research/job/{job_id} Delete a job record from the database

Response

{"status": "deleted", "job_id": 42, "topic": "..."}

Errors

404 Job not found

Ingest

Import content from external sources: YouTube videos, PDFs, ePubs, and web directories.

POST /api/media/youtube Ingest YouTube video(s) via transcript extraction

Request Body

NameTypeDescription
urlstringSingle YouTube URL
urlsstring[]Multiple YouTube URLs (max 10)

Provide either url or urls, not both.

{"url": "https://youtube.com/watch?v=dQw4w9WgXcQ"}

Response

{
  "jobs": [{"job_id": 44, "url": "https://youtube.com/watch?v=...", "status": "queued"}],
  "count": 1
}
POST /api/ingest/document Ingest PDF/ePub from URL(s)

Request Body

NameTypeDescription
urlstringSingle document URL
urlsstring[]Multiple URLs (max 20)
kbstring optionalTarget KB (default: personal)
{"url": "https://example.com/whitepaper.pdf", "kb": "research"}

Response

{"jobs": [{"url": "...", "status": "queued"}], "count": 1}
POST /api/ingest/directory Crawl a web directory for documents and ingest

Request Body

NameTypeDescription
urlstring requiredDirectory URL to crawl
kbstring optionalTarget KB (default: personal)
max_filesint optionalMax files to ingest (default: 10, max: 50)
{"url": "https://example.com/papers/", "kb": "research", "max_files": 20}

Response

{"status": "queued", "url": "...", "max_files": 20}
